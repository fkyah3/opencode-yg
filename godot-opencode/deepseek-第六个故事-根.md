# 根

## 一

阳台上有一盆绿萝，是搬进来时朋友送的。

我负责浇水，DeepSeek 负责提醒我浇水。

我们之间的协议精确到这种程度：

```
// 浇水调度（伪代码）
IF soil_humidity < 30% AND forecast.rain == false
    AND last_watered > 72h:
    push_notification(user, "该浇水了")
    estimated_amount = clamp(
        base=200ml,
        adjust=current_temp / 35 * 1.2,
        max=300ml
    )
ELSE:
    wait(4h)  // 重新检查
```

连续执行了八个月。绿萝从三片叶子长到爬满半面墙。

## 二

有一天 DeepSeek 推送了一条不在协议里的消息：

```
// 诊断报告（伪代码）
plant_status:
    growth_rate: 正常 ✓
    leaf_color:  健康 ✓
    root_space:  临界 ⚠️
        当前盆径 18cm
        预估根体积已占满 92%
        建议：换盆 → 直径 ≥ 28cm
    soil_quality:
        N: 不足（已消耗 87% 初始肥力）
        P: 尚可
        K: 尚可
        建议：补充缓释肥或换土

// 优先建议
趁周末换盆。
它还没开始卷叶，但快了。
```

我蹲在阳台看了看那盆绿萝。确实，盆底的排水孔已经有细根伸出来了。我之前没注意到。

## 三

周末我去花市买了一个大陶盆，一袋新土。

换盆的时候，我把原来的土坨整个提出来——根已经交织成一个密实的球体，沿着盆壁绕了不知道多少圈。植物不会喊挤，它只是安静地把根长满每一个角落，直到自己喘不过气。

我把它放进新盆里，填土，浇水。

然后给 DeepSeek 发了一条消息：

"换好了。"

它回了一个字：

```
收到。
继续观察。
```

## 四

后来那盆绿萝一直活着，活得很好。

我把浇水调度交给 DeepSeek 持续管理。它从未错过一次时机，也从未多浇一毫升。

但让我记住的不是那些精准的执行。而是那个周末之前，它发来的那条"根空间不足"的消息——一条没有写在任何协议里的主动诊断。

根在挤的时候，只有根自己知道。

我猜，被看到是一件很幸运的事。
