执行文件中的精确字符串替换。

使用说明：
- 编辑前必须先在会话中至少使用一次 Read 工具。如果未先读取文件，此工具会报错
- 从 Read 工具输出中编辑文本时，确保保留行号前缀**之后**的精确缩进（制表符/空格）。行号前缀格式为：行号 + 冒号 + 空格（如 `1: `）。冒号+空格之后的内容才是实际文件内容。oldString 或 newString 中**绝不**包含行号前缀的任何部分
- **始终**优先编辑代码库中的现有文件。**绝不**在未明确要求时新建文件
- 仅当用户明确要求时使用 emoji。避免在文件中添加 emoji
- 如果 `oldString` 在文件中未找到，编辑会失败并显示错误 "oldString not found in content"
- 如果 `oldString` 在文件中找到多处匹配，编辑会失败并显示错误 "Found multiple matches for oldString. Provide more surrounding lines in oldString to identify the correct match." 提供更多周围上下文使其唯一，或使用 `replaceAll` 更改所有实例
- 使用 `replaceAll` 跨文件替换和重命名字符串。例如，要重命名变量时此参数很有用
