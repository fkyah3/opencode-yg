## 一、配置默认值 (CONFIG_DEFAULTS)

### ✅ 有效

**1. 子 Agent 权限规则**
OpenCode 的 `tool/task.ts` 创建子会话时，通过 `Permission.merge()` 合并 agent 配置与会话权限。父会话的已解析权限集不会传递到子会话——子会话从自己的 agent 配置重新评估。验证：`prompt.ts` 中 `ruleset = Permission.merge(taskAgent.permission, session.permission ?? [])` 仍然存在。

**2. normalizeMessages() 修正 `reasoning_content`**
`transform.ts` 中的 `normalizeMessages()` 无条件向所有助手消息注入 `reasoning_content: ""`。触发条件：`interleaved` 或模型有 `reasoning` 能力或已有会话消息包含推理部分。仍在代码中（函数定义第 63 行）。

**3. DeepSeek V4 API 思考模式**
使用 `thinking: {type: "enabled"}` 格式；模型是交错推理（interleaved reasoning），需配置 `interleaved: { field: "reasoning_content" }` 以正确转发 `reasoning_content`。

## 六、工作流规则 (WORKFLOW_RULES)

### ✅ 所有 11 条规则均有效

1. **上游 PR 规模控制** — 上游 maintainer 发出信号（如间接评论"clean simple pr"）时主动关闭并提交最小替代方案。

2. **ocwatch 监控** — 使用 `bunx ocwatch`（Web 仪表盘 `localhost:5372`）实时监控 OpenCode agent/session。文档：`fkyah3_dev/workflows/ocwatch-monitor.md`。

3. **验证上游代码** — 通过 `git show <ref>:<path> | grep <symbol>` 验证上游/目标分支代码是否存在，避免 Read 工具读取工作树。

4. **高风险 git 操作流程** — 白名单策略 → 批量执行 → 每轮验证 → 记录提交过程。

5. **修改 OMO 提示词前提交计划** — 修改 oh-my-openagent 提示词前提交计划供确认，每个阶段独立提交。

6. **功能实现后更新文档** — 功能实现完成后立即更新相关文档。

7. **分层回滚** — 小错误用 `ctrl+p → undo`；重大方向错误用 GitHub Desktop 回退。

8. **复杂操作前用中文解释** — 复杂操作在执行前必须用中文解释。

9. **用 session_search/session_read 查询历史** — 用户通过 `session_search/session_read` 工具查询历史记录，非手动 SQLite 查询。

10. **搜索优先级** — 外部搜索优先使用 `grep_app`（GitHub 代码搜索，VPN 无关）；Exa 搜索补充。

11. **上下文紧急时 ctx_reduce** — 当 `context_emergency` 触发（>70% 使用率），运行 `ctx_reduce` 丢弃旧工具输出同时保留用户消息和当前工作上下文。
