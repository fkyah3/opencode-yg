## 编号 001 — 仓库路径迁移后 snapshot 报错

### 场景

将 opencode 仓库从 `E:\fkyah3\Agent\deepseek\opecode\opencode-fkyah3` 复制到 `E:\agent\opencode-fkyah3`，所有配置文件路径全部更新后，启动 opencode 正常，但日志出现：

```
WARN service=snapshot exitCode=128 
fatal: Unable to create '.../index.lock': File exists.
Another git process seems to be running...
failed to add snapshot files
INFO service=snapshot hash= cwd=E:\fkyah3\...\opencode-fkyah3
```

### 根因

opencode 的 **snapshot 子系统** 使用 git 来做增量快照缓存。当仓库路径迁移后：

1. **旧路径仓库仍然存在** —— 复制操作只是拷贝，没有删除旧仓库
2. **snapshot 缓存仍指向旧路径** —— snapshot 在 `C:\Users\<user>\.local\share\opencode\snapshot\<hash>\` 下的 git 仓库中缓存了旧路径的 `cwd`
3. **残留 index.lock 文件** —— 旧路径上某次 snapshot 操作崩溃后留下了锁文件
4. snapshot 启动时尝试在旧路径操作 git → 碰到锁 → `exitCode=128`

### 症状

- 启动正常，功能正常
- 日志文件中有持续的 WARN 信息
- 在写操作后可能出现短暂卡顿（snapshot 重试回退）
- snapshot 功能持续失败但不致命

### 修复

```powershell
# 1. 删除 snapshot 缓存中的残留锁文件
Remove-Item "C:\Users\<user>\.local\share\opencode\snapshot\<hash>\<subdir>\index.lock"

# 2. 或直接清空整个 snapshot 缓存目录（下次启动自动重建）
Remove-Item "C:\Users\<user>\.local\share\opencode\snapshot\*" -Recurse

# 3. 如果旧路径仓库不再需要，可以删除旧目录
Remove-Item "E:\<old-path>\opencode-fkyah3" -Recurse
```

### 预防

仓库迁移后，记得：
1. 重启 opencode（用新路径启动）
2. 清理 snapshot 缓存
3. 删除旧路径仓库（避免两边 git 操作冲突）

## 编号 002 — 子代理 spawn 失败：Unauthorized

### 场景

调用 `task()` 创建子 agent 时，抛出错误：

```
Subagent spawn blocked: failed to resolve session lineage for ses_xxx,
so background_task.maxDepth cannot be enforced safely. Unauthorized
```

### 根因

调用链：

```
task() → omo BackgroundManager → client.session.get(sessionID)
        → SDK 构建 HTTP 请求 → Hono Server.app.fetch()
          → AuthMiddleware.basicAuth() 拦截
          → 401 Unauthorized ← 插件 client 没带认证头
```

Server 总是有密码（或自动生成密码），但 oh-my-openagent 插件内部的 HTTP client 没有传 `Authorization: Basic ...` 请求头。

### 修复

在 `plugin/index.ts` 的 fetch 参数中注入 `x-opencode-internal` 请求头，然后在 `server/middleware.ts` 中识别该请求头跳过 basicAuth：

```typescript
// plugin/index.ts
fetch: async (...args) => {
  const req = new Request(...args)
  req.headers.set("x-opencode-internal", "true")
  return (await Server.Default()).app.fetch(req)
},

// server/middleware.ts
export const AuthMiddleware = (c, next) => {
  if (c.req.header("x-opencode-internal")) return next() // 跳过内部请求
  // ... 正常 basicAuth 逻辑 ...
}
```

### 适用版本

opencode-fkyah3 所有版本（从 fork 开始就存在，但某些场景下 `resolveSubagentSpawnContext` 不一定会触发，所以不一定能复现）。

## 编号 003 — magic-context.jsonc 配置值无效

### 场景

启动后警告：

```
⚠️ Magic Context Config Warning
Some configuration values are invalid and were replaced with defaults:
- [user config] "execute_threshold_percentage": invalid value (number 90), using default 65.
```

### 根因

`magic-context.jsonc` 中的 `execute_threshold_percentage` 被设置为 `90`，但 magic-context 的有效值范围是 `0-65`。配置校验通过 Zod schema，超范围的值被触发默认值。

### 修复

删除或修改该字段：

```jsonc
// magic-context.jsonc 中
"execute_threshold_percentage": 65,
// 或直接删除该字段（回退到默认值 65）
```

## 编号 004 — 路径迁移后 `opencode` 命令失效

### 场景

将仓库从旧路径复制到新路径后，在终端输入 `opencode` 仍然启动旧路径的版本，或报模块找不到。

### 根因

`opencode` 命令通过某个注册文件（`.cmd` 文件或 PATH 中的脚本）指向硬编码的路径。迁移后该路径未更新。

### 检查方法

```powershell
# 找到 opencode 命令的注册位置
Get-Command opencode | Select-Object Source
```

通常会指向：
- `C:\Users\<user>\bin\opencode.cmd`
- 或类似自定义 PATH 目录

### 修复

打开该文件，将其中的旧路径替换为新路径。

## 编号 005 — OH MY OPENCODE plugin 路径指向旧仓库

### 场景

在 `opencode.json` 中，plugin 配置为：

```json
"plugin": [
    "file:///E:/<旧路径>/oh-my-openagent",
    "file:///E:/<旧路径>/opencode-magic-context/packages/plugin"
]
```

仓库迁移后，这些绝对路径仍然指向不存在或过时的位置。

### 修复

```jsonc
"plugin": [
    "file:///E:/<新路径>/plugins/oh-my-openagent",
    "file:///E:/<新路径>/plugins/opencode-magic-context/packages/plugin"
]
```

## 规则总结：仓库迁移后必须检查的 5 项

| # | 检查项 | 说明 |
|:-:|--------|------|
| 1 | `opencode.cmd` | PATH 中注册的启动脚本 |
| 2 | `opencode.json` | 插件路径（2 处） |
| 3 | snapshot 缓存 | 删除旧缓存中的 index.lock |
| 4 | 旧路径仓库 | 确认是否要删除 |
| 5 | 当前运行的实例 | 检查是否还有旧实例在运行 |
