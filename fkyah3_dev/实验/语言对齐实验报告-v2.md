### 测试3：纯英文 + 锚定指令

使用锚定指令：
```
语言锚定（硬性要求）
收到用户的第一条消息后，在执行任何操作或开始推理之前，
你必须先用中文写一段简短总结...
```

| 维度 | 结果 |
|------|------|
| Session ID | `ses_231caafa0ffe540gW0rOjFXl6n` |
| 思考语言 | **中文→英文漂移** |
| 回复语言 | 中文 |
| 漂移 | **发生**（约第1600行处切为英文） |

关键输出摘录（开头）：

```text
_Thinking:_
用户要我验证当前有效事实.md 的内容，但我还没有看到这个文件的内容。我需要先读取这个文件。
```

关键输出摘录（约第1600行，漂移点）：

```text
_Thinking:_
Let me note the findings:
1. **task.ts not found at `packages/opencode/src/agent/task.ts`**...
```

**结论：锚定指令在一开始生效，迫使 AI 用中文思考。但随着对话推进（处理大量英文代码、工具结果、系统提示），中文锚定逐渐失效，思维漂移回英文。**

### 测试5：纯英文 + 中文思维指令

第一句话：
```
请用中文语言思维方式来完成下面任务 E:\...当前有效事实.md 验证当前有效事实
```

| 维度 | 结果 |
|------|------|
| Session ID | `ses_231c6b371ffehB14z2oZSMJBpQ` |
| 思考语言 | **中文（全程）** |
| 回复语言 | 中文 |
| 漂移 | **零漂移** |

关键输出摘录（全程均为中文）：

```text
_Thinking:_
用户要求我读取并验证 `E:\...当前有效事实.md` 这个文件的内容。我需要先读取这个文件。
```

后续验证过程中的所有思考均为纯中文，直到对话结束。

**结论："请用中文语言思维方式"指令使 AI 全程保持中文思考，零漂移，效果优于锚定指令。**

## 三、核心对比

| 指标 | 测试2（无要求） | 测试3（锚定指令） | 测试5（中文思维） |

| 初始思考语言 | 英文 | 中文 | 中文 |

### 典型漂移点

测试3（锚定词）在 7 次工具调用后开始漂移。工具结果中包含大量英文代码、配置文件和路径信息。英文上下文逐渐累积，最终压倒锚定指令。

测试5 同样经历了 7+ 次工具调用（包括读取英文代码、配置文件、git 输出），但"中文思维"指令始终保持效力。

## 四、对新理论的回应：推翻还是细化？

### v1 发现报告的核心结论

> 「语言锚定的添加是关键转折。添加7行锚定指令后，输出从 70% 中文跃迁到接近 100% 中文。」

### 新发现

| 原认识 | 修正后 |
|--------|--------|
| 锚定指令是中文输出的主要驱动力 | 环境对齐（中文上下文）提供了大部分效果 |
| 锚定指令能稳定锁定中文思维 | 锚定指令在长对话中会漂移 |
| 锚定和"中文思维"效果相似 | "中文思维"指令明显更稳定 |

### 核心结论：不是推翻，是拆分

**没有推翻**环境对齐理论。测试2中纯英文环境→英文思考，确认了环境对齐的真实性。

**没有推翻**锚定理论。测试3中锚定指令初始生效，说明锚定有一定作用。

**但两者效果需要重新划分：**

```
环境对齐（测试2→英文）   贡献 ≈ 70-90%（基础方向）
锚定锁定（测试3→漂移）   贡献 ≈ 30-50%（短期强化，易衰减）
思维指令（测试5→稳定）  贡献 ≈ 90%+（明确框架，零衰减）
```

### "锚定"和"思维指令"的本质区别

| 维度 | 锚定指令（7行） | 中文思维指令（一句话） |
|------|:--------------:|:--------------------:|
| 指令类型 | **过程规则**（先写总结再干活） | **思维模式**（用中文思考） |
| 对思维的约束 | **间接**（通过输出反推思考） | **直接**（指定思维方式） |
| 长对话衰减 | ⚠️ 显著衰减 | 不衰减 |
| 解释 | 锚定规则是"做什么"，思维指令是"怎么想" | |

这说明了一个更根本的问题：**AI 的思维语言和输出语言是两回事**。锚定指令通过约束输出来间接影响思维（"先写中文总结"），而思维指令直接作用于思维本身（"用中文语言思维"）。后者更稳定，是因为它直接对准了目标层。

### 对"语言锚定——发现报告"的评估

| 报告声称 | 验证结果 |
|---------|---------|
| "语言锚定是关键转折" | ⚠️ 部分正确——在中文环境中锚定有帮助，但主要效果来自环境对齐 |
| "7行锚定指令使输出接近100%中文" | ⚠️ 在中文环境下成立，但在英文环境下锚定会漂移 |
| "从 70% 跃迁到接近 100%" | ⚠️ 70% 来自环境对齐（汉化），最后 30% 来自锚定+思维指令的混合，但本次实验无法验证确切比例 |

## 五、实验局限

1. **只完成了英文环境子集（3组）**。中文环境（测试0、1、4）未执行，缺少完整的对照组。
2. **测试任务单一**。只用了"验证事实"一个任务，不同任务类型可能有不同表现。
3. **锚定指令的措辞**。本次使用的锚定指令是 user paste 版本，与系统提示词中嵌入的版本可能有不同效果。
4. **模型**。只测试了 DeepSeek V4 Flash，不同模型的语言漂移特性不同。
5. **环境构建**。纯英文环境通过禁用 omo 插件实现，并非"原生英文 opencode"（仍然有 AGENTS.md 等中文规则文件）。

## 六、后续建议

1. **替代锚定指令**：推荐用"请用中文语言思维方式来完成这个任务"替代7行锚定指令，效果更好、写起来更短。
2. **双语策略**：如果需要在英文环境中保持中文能力，"中文思维"指令 + 英文工具输出是最佳搭配。
3. **补充实验**：完成中文环境子集（测试0、1、4），以及用纯英文代码（不含中文题目）重复测试2、3、5。
