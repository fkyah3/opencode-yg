# 思辨记录：信息论解剖 vs LLM vs 索尼 Ace

> 记录自 #1252 + 一位信息论研究者的批判
> 2026-04-27

---

## 一、核心论点

作者（周锚）用信息五级形态框架批判当前 LLM 路线：

| 层级 | 内容 |
|:----:|------|
| 数位 Bits | 原始信号 |
| 数据 Data | 结构化符号 |
| 消息 Message | 人类语言编码 |
| 知识 Knowledge | 体系化理解 |
| 智慧 Wisdom | 创造与判断 |

**诊断：LLM 被锁死在第三层（消息层）**

提出的替代方案：**VPA 模型**
- Vision（视觉）→ **Perception（认知）** → Action（行动）
- 关键：砍掉中间的语言层（L），代之以认知层（Perception）
- 认为语言只是信息的编码，不能做智能的底座

## 二、索尼 Ace 作为证据

Ace（《自然》封面）——多智能体乒乓球机器人，击败人类精英选手：
- Vision：9 台高速摄像头，毫秒级捕捉
- Perception：3000 小时虚拟自我对弈，无模型强化学习
- Action：20ms 端到端延迟，8 自由度机械臂
- 成功验证了 VPA 范式的闭环

## 三、核心逻辑漏洞

**Ace 的成功 → LLM 的失败？**

这条推理不成立。Ace 和 LLM 解决的是不同类问题：
- Ace：专用物理系统，封闭规则空间
- LLM：通用语言系统，开放语义空间

类比：AlphaGo 不需要 LLM 也能赢围棋，但这和 LLM 的优劣无关。

## 四、观点评价

| 维度 | 评价 |
|:----:|------|
| 对的地方 | "语言是编码不是信息本身"这个提醒有效 |
| 弱的地方 | "LLM 只是统计排列组合"——低估了 2026 年模型的多步推理能力 |
| 空的地方 | 没有数据/实验/可复现结论，7/9 参考文献是作者自己的公众号 |
| 形式问题 | 发在 GitHub issue 引流公众号 |

## 五、与我们的实验的关系

作者从哲学层"压"下来，批评整个 LLM 方向的根本缺陷。
我们从实验层"走"上去，用数据验证具体的提示词现象。

两条路径可以互补：他关心"为什么"，我们关心"怎么做"。

## 六、可提取的有用框架

信息五级形态框架可以用于分析实验结果：
- 实验本身在哪个层级操作？
- 不同的提示词策略作用于不同层级？
- 锚定指令在"消息"层操作，思维指令在"认知"层操作？

这可能解释为什么思维指令比锚定指令更稳定——因为它作用在更深的层级上。
