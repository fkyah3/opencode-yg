# 新手分发方案

## 目标

下载一个包，解压即用。新手只需做 3 件事：
1. 解压到任意目录
2. 放 API Key
3. 双击启动

## 分发包结构

```
opencode-fkyah3-windows-x64-v2.0.zip/
├── bin/
│   └── opencode.exe
├── plugins/                          ← 预装插件，解压即生效
│   ├── magic-context/
│   └── ... (其他要预装的)
├── opencode.json                     ← 预配置：DeepSeek V4 Flash、中文等
├── 首次启动.bat                      ← 一键脚本：放 key → 注册环境变量 → 启动
└── 使用说明.txt
```

## Key 处理方案

### 方案 1：环境变量（推荐）

用户运行 `首次启动.bat`，脚本提示粘贴 Key，写入环境变量：

```bat
:: 首次启动.bat
@echo off
set /p KEY=请输入你的 DeepSeek API Key:
setx DEEPSEEK_API_KEY "%KEY%"
echo 环境变量已设置，启动 opencode...
bin\opencode.exe
```

`opencode.json` 中引用环境变量（**分发包预制，永远不改**）：

```json
{
  "providers": {
    "deepseek": {
      "apiKey": "$DEEPSEEK_API_KEY"
    }
  }
}
```

**优点：** 零实现成本，opencode 原生支持环境变量
**缺点：** 环境变量作用域有限，换终端可能不生效

### 方案 2：.env 文件（备选）

`首次启动.bat` 把 Key 写到包内的 `.env` 文件：

```bat
@echo off
set /p KEY=请输入你的 DeepSeek API Key:
echo DEEPSEEK_API_KEY=%KEY% > .env
echo Key 已保存到 .env，启动 opencode...
bin\opencode.exe
```

**优点：** 随包移动，不需要系统级注册
**缺点：** AI 帮忙配置时可能误读到 Key

### 方案 3：内嵌到 opencode.json（不推荐）

```bat
@echo off
set /p KEY=请输入你的 DeepSeek API Key:
powershell -Command "(Get-Content opencode.json) -replace '\"YOUR_KEY_HERE\"', '\"%KEY%\"' | Set-Content opencode.json"
echo Key 已配置到 opencode.json，启动 opencode...
bin\opencode.exe
```

**缺点：** 配置分享、git 提交都会带出 Key，AI 帮忙改配置时也可能误操作

## 推荐方案：环境变量

**Key 放在环境变量中，AI 永远不会看到它。**

```
Key → 系统环境变量（setx / .bashrc）
         ↓ AI 不可见
opencode.json 引用 $DEEPSEEK_API_KEY
         ↓ AI 可见可改，但 Key 不在这里
AI 帮忙改配置时，改不到 Key，泄漏不了
```

### 为什么只有环境变量

| 方案 | AI 能看到 Key | 泄露风险 | 核心问题 |
|------|-------------|---------|---------|
| 环境变量 | ❌ 不可见 | 低 | — |
| `.env` 文件 | ✅ 可见 | 中 | AI 帮配东西时可能读取或误改 |
| 写入 `opencode.json` | ✅ 可见 | 高 | 配置分享、git 提交都会带出去 |

**核心逻辑：** AI 帮忙配置时，如果它连 Key 都能看到，哪天脑子抽了把配置发出去，Key 也跟着泄了。Key 放在 AI 可见范围之外才安全。

## 如果想做到极致：方案 B+

在方案 B 基础上，可以写一个 `setup.ts` 脚本，用 Bun 编译成一个小 exe（`配置工具.exe`），提供交互式界面：

```
配置工具.exe
  │  1. 输入 DeepSeek API Key（掩码输入，不显示明文）
  │  2. 选择模型（Flash / Pro）
  │  3. 测试连接（发一个请求验证 Key 可用）
  │  4. 确认 → 写入配置 → 启动 opencode
```

但这对新手来说可能过于复杂——**bat 脚本是最低门槛。**
