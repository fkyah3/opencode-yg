This is not an isolated issue — it's a systemic compatibility problem affecting multiple open-source projects using DeepSeek V4 thinking mode + tool calls. Same root cause, same error across all of them.

**Known affected projects (as of April 26):**
| Project | Issue |
|---------|-------|
| OpenCode | #24104 #24130 #24261 |
| OpenClaw | openclaw#71160 openclaw#71455 |
| n8n | n8n#29119 |
| Continue.dev | continuedev#10758 |
| HuggingFace Chat-UI | chat-ui#1664 |

**Root cause:** AI SDKs (@ai-sdk/openai-compatible, OpenRouter gateway) don't preserve reasoning_content on assistant messages with tool_calls across turns. DeepSeek V4 thinking mode requires it on every assistant message after a tool call — including empty strings.

**Fix direction (verified across projects):** Unconditionally inject reasoning_content: "" for all assistant messages before API dispatch. Our fork (fkyah3/opencode-fkyah3, commit b5b6ad05d) implements this at the normalizeMessages() level. OpenClaw implemented a similar approach in #71105.

This is not a config issue on your side — you don't need to switch providers or models.
